# Lazy VFDT Thesis Project

## Quick Start
1. Activate the `dtree` conda env (already has requirements) or run `pip install -r requirements.txt`.
2. Run comparisons:
   - Synthetic default: `conda run -n dtree python compare_lazy_vs_dwm.py`
   - Electricity: `conda run -n dtree python compare_lazy_vs_dwm.py --dataset electricity --data-path "C:\\...\\elecNormNew.csv" --seeds 13 29 47 83 101 --csv-out results.csv`
   - Gas Sensor (libsvm batches already merged to `gas.csv`): `conda run -n dtree python compare_lazy_vs_dwm.py --dataset gas_sensor --data-path gas.csv --preset gas --scale-per-batch --seeds 13 29 47 83 101 --csv-out results.csv`
   - Airlines: `conda run -n dtree python compare_lazy_vs_dwm.py --dataset airlines --data-path "datasets\\airlines.arff\\airlines.csv" --scale --ldt-max-depth 10 --ldt-grace-period 20 --dwm-base-depth 4 --dwm-window-size 120 --seeds 13 29 47 83 101 --csv-out results.csv`
3. After adding runs, regenerate the summary: `conda run -n dtree python tools/summarize_results.py --csv results.csv --out docs/results_summary.md`.

## Repository Map
- `lazy_decision_tree/` – Lazy VFDT implementation (lazy pruning + Hoeffding bound).
- `dwm/` – Dynamic Weighted Majority baseline.
- `compare_lazy_vs_dwm.py` – main experiment driver (supports synthetic + Electricity + Gas Sensor + Airlines; multi-seed + CSV logging).
- `datasets/` – loaders for electricity, gas_sensor, airlines (+ conversion utility in `tools/prepare_gas_sensor_csv.py`).
- `results.csv` – append-only log of experiment runs.
- `docs/results_summary.md` – aggregated metrics (generated by `tools/summarize_results.py`).
- `docs/report_prep.md` – notes on dataset coverage, preprocessing, and recommended thesis outline.
- `docs/thesis_scope.md` – locked scope, research questions, and status references.
